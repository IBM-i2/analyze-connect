    login

    Logs in to a service by passing the specified credentials to a URL that was specified in authentication configuration.

    When a service is configured to require authentication, the i2 Connect gateway uses its login endpoint, passing the credentials that a user provided.

    If authentication succeeds, the response contains a token that the gateway uses in calls to other service endpoints. If it fails, the response contains information in RFC 7807 format that the gateway can use to provide information to the client.

    login

    Request
    POST /{loginUrl}
    Parameters
    Name In Type Default Notes
    *credentials body object

    The credentials for authenticating with the external data source, as defined in the authentication config for the service. For example, if the config is like this:

    "fields": [
      {
        "id": "username",
        "label": "Username",
        "type": "text"
      },
      {
        "id": "password",
        "label": "Password",
        "type": "password",
      }
    ]

    Then the credentials object might be like this:

    {
      "username": "Imogen",
      "password": "thisismypassword"
    }
    Responses
    Status Code Type Description Samples
    200 AuthResponse

    The POST method completed successfully.
    401 ProblemDetails

    Authentication failed because the credentials were not valid.

    Definitions

    AuthResponse

    When a service is configured to require authentication, the i2 Connect gateway uses its login endpoint, passing the credentials that a user provided.

    If authentication succeeds, the response contains a token that the gateway uses in calls to other service endpoints. If it fails, the response contains information in RFC 7807 format that the gateway can use to provide information to the client.

    Name Type Notes
    token string

    A token that can authorize the caller in future requests to any service that uses the same authorization config.

    ProblemDetails

    When a service is configured to require authentication, the i2 Connect gateway uses its login endpoint, passing the credentials that a user provided.

    If authentication succeeds, the response contains a token that the gateway uses in calls to other service endpoints. If it fails, the response contains information in RFC 7807 format that the gateway can use to provide information to the client.

    Name Type Notes
    detail string

    Additional detail about the problem, specific to this instance of it. This information might also be displayed to the user in the form of an error message.
    instance string

    A URL for a resource specific to this instance of the problem. The URL can be absolute or relative to the path of the i2 Connect service.
    status integer (int32)

    The HTTP status code returned by the i2 Connect service.
    title string

    The title of the problem, which might be displayed to the user.
    type string

    A URL for a resource that describes the type of the problem. If the problem is that the service requires authentication, this field must be set to urn:uuid:264caa46-75cb-4ac5-891a-11adeb48b6fb, which triggers the i2 Connect gateway to prompt the user for credentials.
